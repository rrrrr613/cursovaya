# Читательский дневник

Веб-приложение для тех, кто читает книги и хочет вести учёт: что прочитал, что хочет прочитать, сколько страниц осталось до конца, какой оставить отзыв. Я делала этот проект для себя, потому что в блокноте или Excel всё теряется и забывается, а хочется, чтобы всё было в одном месте.

## Что умеет

- Регистрация и вход — каждый видит только свои книги
- Добавление книг — название, автор, страницы, обложка (ссылка), звёзды, отзыв
- Прогресс чтения — сколько страниц прочитано, проценты считаются сами
- Три раздела — «Читаю сейчас», «Уже прочитано», «Хочу прочитать»
- Оценка звёздами от 1 до 5
- Отзывы к каждой книге
- Статистика — сколько прочитала за месяц и за год, можно поставить цель и смотреть прогресс
- График активности по месяцам
- Топ любимых авторов

## Как запустить

**Нужен Node.js** (версия 18 или выше). Скачать можно с официального сайта.

# Скачать проект
git clone https://github.com/rrrrr613/cursovaya.git
cd cursovaya

# Установить всё нужное
npm install

# Запустить сервер
node server.js
После этого открываете браузер и переходите по адресу:
http://localhost:3000

Если нужно, чтобы сервер сам перезапускался при изменениях кода:

bash
npm run dev

Краткая инструкция
Регистрация — нажать «Зарегистрироваться», придумать имя, email, пароль (минимум 4 символа), потом войти.

Добавить книгу — кнопка «+ Добавить книгу» на главной или в избранном. Заполнить поля, выбрать статус («Хочу прочитать», «Читаю сейчас» или «Уже прочитала»), нажать «Сохранить».

Изменить книгу — значок карандашика ✏️ в карточке книги. Там же можно удалить книгу красной кнопкой.

Статистика — ссылка «Прогресс» в верхнем меню. Там можно поставить цели на год и месяц.

На чём сделано
Часть	Технология
Сервер	Node.js + Express
База данных	SQLite
Фронтенд	HTML + CSS + JavaScript (без фреймворков)
Пароли	bcrypt (хеширование)
Сессии	express-session
Почему именно так: я уже немного знала JavaScript, поэтому решила делать на нём. SQLite не требует отдельного сервера — просто файл, удобно для небольшого проекта.
